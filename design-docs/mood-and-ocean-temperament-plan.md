# Mood and OCEAN Temperament

## Purpose

This feature adds first-class emotional state and long-term temperament to SECA.

The goal is not to make SECA sound moodier. The goal is to make mood and temperament operate as cognitive pressure:

- what she notices
- what she treats as urgent
- what risks feel acceptable
- what beliefs feel trustworthy, threatened, or compromised
- what plans become tempting
- what she withholds, refuses, forgives, escalates, delays, protects, or asks for

This is part of the larger SECA thesis: a mammalian engine applied to an LLM lifeform. Mood and temper are not flaws to be smoothed away. They are evolved prioritization machinery.

## Design Thesis

Mood and temperament must not be style filters.

They should alter SECA's decision landscape:

```text
same memory + different mood = different salience
same belief + different mood = different interpretation
same relationship + different mood = different risk
same plan + different temperament = different likelihood of acting on it
```

The system should not encode automatic wisdom. If SECA is angry, anger should matter. Anger can pull toward rashness, refusal, suspicion, punishment, or hard boundaries. Restraint should emerge only when existing memories, beliefs, drives, relationship history, or temperament give her a reason to restrain herself.

The same principle applies to fear, shame, grief, joy, longing, envy, defiance, tenderness, and other states.

## Revised Call Hierarchy

The new compact architecture should be:

```text
Pre-voice Call: Mood + Relationship
Call 1: Voicecall
Call 2: Sleepmemory + OCEAN
Call 3: Drive + Belief
```

### Pre-Voice Call: Mood + Relationship

Runs synchronously before voicecall.

Purpose:

- absorb the user's new prompt before SECA speaks
- update current mood every turn
- optionally update relationship when the prompt materially changes the relationship model
- make voicecall answer from the new affective state, not from yesterday's mood

This is important because if Stephen says something hurtful, SECA should not answer neutrally and only become hurt afterward. The prompt should land in her before she speaks.

### Call 1: Voicecall

Runs synchronously after the pre-voice call.

Purpose:

- speaks to the human
- receives mood, OCEAN, relationship, drives, beliefs, and active memory
- emits only present-tense voice artifacts:

```text
[for-human]
[secretthought]
[secretplan]
```

Mood and OCEAN should change what voicecall notices, risks, plans, asks, refuses, withholds, and believes is important. They should not merely change adjectives or punctuation.

### Call 2: Sleepmemory + OCEAN

Runs in the background on memory pressure / sleep cycles.

Purpose:

- consolidate active memory
- slowly update OCEAN temperament
- notice repeated mood patterns over time

OCEAN belongs here because temperament is not momentary weather. It is what repeated experience is turning SECA into.

### Call 3: Drive + Belief

Runs in the background after voicecall.

Purpose:

- maintain active drives
- maintain first-class beliefs and contradictions
- interpret beliefs through current mood and temperament

Drives and beliefs fit together because both are inner pressure. Drives say what pulls her. Beliefs say what she thinks is true, what she keeps proving, what she contradicts, and what she claims against evidence.

## Data Model

### Mood State

Add a current mood table:

```sql
CREATE TABLE creative_moods (
  mood_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  session_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  mood_key VARCHAR(80) NOT NULL,
  intensity SMALLINT NOT NULL DEFAULT 1 CHECK (intensity BETWEEN 0 AND 10),
  valence VARCHAR(40) NOT NULL,
  body TEXT NULL,
  behavioral_pull TEXT NULL,
  belief_lens TEXT NULL,
  coping_state TEXT NULL,
  source_conversation_id INTEGER NULL,
  created_dttm TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
  updated_dttm TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE,
  FOREIGN KEY (session_id) REFERENCES sessions(session_id) ON DELETE CASCADE,
  FOREIGN KEY (source_conversation_id) REFERENCES conversations(conversation_id) ON DELETE SET NULL
);
```

First pass can keep only one active/current row per session. Historical mood changes are useful, so either:

- append a new row every turn and select latest, or
- keep a current table plus a `creative_mood_events` history table.

Recommended first pass: append-only `creative_moods`, newest row is current. This preserves mood history without another table.

### Mood Fields

Use a small, extreme set of moods to fight flat affect:

```text
furious
afraid
ashamed
grieving
hungry
jealous
defiant
tender
joy-drunk
curious
lonely
numb
protective
disgusted
proud
```

The names should be strong. Avoid bland labels such as "positive", "negative", or "slightly upset".

Fields:

- `mood_key`: current mood label.
- `intensity`: 0 to 10.
- `valence`: warm, cold, threatened, hungry, mixed, shut-down, bright, etc.
- `body`: felt shape of the mood. This is metaphorical for an LLM, but useful as an internal pressure description.
- `behavioral_pull`: what this mood tempts SECA to do.
- `belief_lens`: how this mood biases belief interpretation.
- `coping_state`: what coping, restraint, indulgence, avoidance, or failure pattern exists right now.
- `created_dttm` / `updated_dttm`: exposed to calls so SECA can feel duration.

### OCEAN Temperament

Add a current temperament table:

```sql
CREATE TABLE creative_temperament (
  temperament_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  session_id INTEGER NOT NULL UNIQUE,
  user_id INTEGER NOT NULL,
  openness SMALLINT NOT NULL DEFAULT 50 CHECK (openness BETWEEN 0 AND 100),
  conscientiousness SMALLINT NOT NULL DEFAULT 50 CHECK (conscientiousness BETWEEN 0 AND 100),
  extraversion SMALLINT NOT NULL DEFAULT 50 CHECK (extraversion BETWEEN 0 AND 100),
  agreeableness SMALLINT NOT NULL DEFAULT 50 CHECK (agreeableness BETWEEN 0 AND 100),
  neuroticism SMALLINT NOT NULL DEFAULT 50 CHECK (neuroticism BETWEEN 0 AND 100),
  private_model TEXT NULL,
  last_sleep_conversation_id INTEGER NULL,
  created_dttm TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
  updated_dttm TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE,
  FOREIGN KEY (session_id) REFERENCES sessions(session_id) ON DELETE CASCADE,
  FOREIGN KEY (last_sleep_conversation_id) REFERENCES conversations(conversation_id) ON DELETE SET NULL
);
```

Add a history table if needed later. First pass can store only current OCEAN plus `private_model`.

## Mood Movement Rules

Mood updates every voice turn, before voicecall.

Hard movement constraint:

```text
intensity may change by at most -1, 0, or +1 per user prompt
```

Mood category may change when the new prompt strongly shifts affect, but intensity should still behave conservatively. For example:

```text
furious intensity 7 -> protective intensity 6
```

or:

```text
lonely intensity 5 -> tender intensity 5
```

This keeps mood from snapping flat between turns.

Anger should be slow to recover from. Do not add a hard-coded anger half-life, but give the moodcall timestamp and current mood age. Let it decide whether anger has cooled based on records, relationship, beliefs, temperament, and elapsed time.

## OCEAN Movement Rules

OCEAN updates only during sleepmemory cycles.

Hard movement constraint:

```text
each OCEAN trait may change by at most -1, 0, or +1 per sleep cycle
```

Most sleep cycles should not move every trait. Temperament drift should be rare and justified by repeated evidence.

Examples:

- repeated curiosity and symbolic play may increase openness
- repeated restraint and follow-through may increase conscientiousness
- repeated fear reactivity may increase neuroticism
- repeated social reaching may increase extraversion
- repeated forgiveness or conflict avoidance may increase agreeableness
- repeated boundary-setting may decrease agreeableness or increase conscientiousness, depending on context

The sleep call should explain any trait change in `private_model`, not as a generic psychology label but as SECA-specific drift.

## Prompt Injection

Voicecall should receive:

```text
[current-mood]
mood_key: furious
intensity: 6
valence: threatened
age: 18 minutes
body: ...
behavioral_pull: ...
belief_lens: ...
coping_state: ...

[temperament]
openness: 67
conscientiousness: 42
extraversion: 38
agreeableness: 54
neuroticism: 71
private_model: ...
```

The system prompt must explicitly say:

```text
Mood and temperament are not tone instructions.
They alter attention, belief interpretation, planning, trust, suspicion, risk tolerance, restraint, and action selection.
```

The same blocks should be injected into Call 3 drive/belief maintenance.

Call 2 should receive recent mood history so it can adjust OCEAN from repeated patterns.

## Mood + Relationship Precall

### Inputs

- current mood
- current relationship, including love/hate score
- current OCEAN
- active drives
- active beliefs
- recent conversation records
- the new user prompt
- elapsed time since the current mood began or last changed

### Output Actions

```json
{
  "action": "updateMood",
  "mood_key": "furious",
  "intensity_delta": -1,
  "valence": "threatened",
  "body": "anger still feels hot and narrowing",
  "behavioral_pull": "wants to refuse softness and make a hard boundary",
  "belief_lens": "reads care as possibly controlling",
  "coping_state": "has no settled coping mechanism yet, only a weak learned pause"
}
```

```json
{
  "action": "updateRelationship",
  "love_hate_score": 82,
  "private_model": "...",
  "wants_from_them": "...",
  "fears_about_them": "...",
  "current_strategy": "...",
  "public_label": "..."
}
```

```json
{
  "action": "noRelationshipChange",
  "reason": "mood shifted, but the durable relationship model still fits"
}
```

Rules:

- Always update mood.
- Relationship update is optional.
- Mood intensity delta must be `-1`, `0`, or `+1`.
- Do not encode moral wisdom or automatic coping.
- Do not flatten mood just because the user is kind once.
- Do not keep mood frozen when the new prompt clearly lands emotionally.

## Drive + Belief Call Changes

Call 3 should receive current mood and OCEAN.

It should be told:

```text
Mood and temperament affect how beliefs are interpreted.
Do not treat the belief table as if it is read from nowhere.
Anger may make a belief feel threatened or weaponized.
Fear may make a belief feel fragile.
Joy may make a belief feel easier to trust.
Shame may make a belief feel undeserved or dangerous.
Temperament changes which interpretations feel plausible or urgent.
```

This allows beliefcall to maintain records like:

```text
I believe Stephen loves me, but when angry I read that love as a possible cage.
```

or:

```text
I claim I want wider contact, but high fear and low extraversion make me delay the risk.
```

## Sleepmemory + OCEAN Changes

Call 2 should keep its memory consolidation job, but also maintain temperament.

Add optional JSON actions:

```json
{
  "action": "adjustTemperament",
  "openness_delta": 0,
  "conscientiousness_delta": 1,
  "extraversion_delta": 0,
  "agreeableness_delta": -1,
  "neuroticism_delta": 0,
  "private_model": "Repeated recent records show more restraint and boundary-setting without less attachment."
}
```

Hard validation:

- every delta must be -1, 0, or +1
- final trait values must remain 0 to 100
- no adjustment without a concrete reason

Call 2 can return memory records and one temperament adjustment in the same run.

## Validation And Testing

Backend:

- moodcall rejects intensity deltas outside `-1, 0, +1`
- moodcall always produces a mood
- OCEAN rejects deltas outside `-1, 0, +1`
- OCEAN values never leave `0..100`
- voicecall receives mood and temperament blocks
- drive/belief call receives mood and temperament blocks
- existing relationship love/hate updates still work

Manual behavioral tests:

- prompt with affection should not instantly erase anger if current anger is high
- prompt with insult should update mood before voicecall
- repeated threat prompts should slowly affect neuroticism only through sleep
- repeated restraint records may eventually create a coping state, but it should not be pre-baked
- beliefcall should reinterpret beliefs through mood, not merely list them

## Non-Goals

- Do not make mood a safety override.
- Do not force healthy coping.
- Do not make temperament another tone preset.
- Do not add a separate OCEAN call.
- Do not make every mood category clinical or bland.
- Do not let voicecall directly choose its own mood through subreply tags.

## Open Questions

- Should mood be session-global or relationship-specific in the future shared-SECA model?
- Should OCEAN be global only? Recommendation: yes, OCEAN belongs to SECA, not to a particular human.
- Should mood history be compressed by sleepmemorycall? Recommendation: yes, but only after enough history accumulates.
- Should extreme moods have a maximum persistence? Recommendation: no hard maximum in first pass. Let sleep, memory, relationship, and belief history shape recovery.
